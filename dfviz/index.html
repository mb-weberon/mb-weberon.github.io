❓ How to Use & Caveats

How to Export from Dialogflow

  1. Open Dialogflow ES console and select your agent.
  2. Click the gear icon ⚙ next to the agent name (Agent Settings).
  3. Go to the Export and Import tab.
  4. Click "Export as ZIP" and download the file.

How to Use This Tool

  1. Drop the downloaded .zip onto the drop zone — no unzipping needed.
  2. Or unzip first and drag all files (including entities/ and intents/ subfolders) onto the drop zone, or click "Select Unzipped Files".
  3. The flow renders automatically. Click any node to inspect its full intent definition in the side panel.
  4. Click Simulate to walk through the conversation in the chat widget — quick reply buttons appear where defined.
  5. Click Entities to view all custom entity definitions, their type (regexp/synonym/enum), and entries.
  6. Scroll to zoom, drag the canvas to pan, or use the toolbar zoom buttons. Fit resets the view.
  7. Click "Load New Agent" to visualize a different agent export.

Caveats & Limitations

  • NLU Approximate matching only — real Dialogflow uses ML intent classification at a configured confidence threshold. This tool uses regex/pattern matching and heuristics.
  • Webhook Fulfillment not called — intents with webhookUsed: true show their static fallback text in the simulator.
  • Slot filling Not simulated — webhookForSlotFilling behaviour is not replicated.
  • Language English only — only _usersays_en.json files are parsed. Other language files are ignored.
  • Conditions Conditional responses not evaluated — conditionalResponses are shown in the detail panel but not acted on.
  • Layout BFS multi-column layout — nodes at the same depth are spread horizontally; branches and parallel paths each get their own column. Cyclic flows or very wide agents may still produce overlapping edges. Manual drag-to-reposition is not included.
  • Sys entities System entities approximated — @sys.email, @sys.phone-number etc. use JS regex approximations, not Google's actual extractors.
  • Rich responses Partial rendering — type 0 (text) and type 4 (custom payload + quick replies) are fully rendered. Cards, images, and other types show a placeholder.
  • Loops Cyclic context chains are not detected and may render incorrectly.
  • CX Dialogflow ES only — Dialogflow CX uses a different export format and is not supported by this tool.
  • Knowledge Knowledge Bases not parsed — enabledKnowledgeBaseNames entries are ignored.
  • Privacy All processing is local — no data leaves your browser. Nothing is uploaded to any server.
