🛠️ Testing & Replay Tools

📋 Live Session Trace (Output)

This area records every valid action you take in the chat. It generates a clean JSON array (e.g., ["user@test.com", "1", "2"]).

  • 💾 Save .json: Click this to download your current path as a file for future testing.
  • Manual Copy: You can also copy the text directly from the dashed box.

📥 Replay Command (Input)

The "Steering Wheel" of the engine. Paste a previously saved JSON array here and click ▶️ Run.

Pro Tip: Replay uses the exact same validation logic as manual typing. If your script contains an invalid email or a non-existent choice, the replay will halt and show an error in the chat history.

🏗️ Configuration Architecture (config.json)

This engine is a Finite State Machine (FSM), closely following the SCXML (State Chart XML) standard for declarative logic.

Root Components

  • initial: The ID of the starting node.
  • states: A dictionary where each key is a unique State ID.

State Definition Table

Type Description Logic Path
Interactive Wait for user text or button click. Follows onValid or choices[].next
Logic Gate Automatic branch based on storeKey data. Uses map to find the next state.

Schema Example

{
  "initial": "ask_email",
  "states": {
    "ask_email": {
      "message": "Enter email:",
      "inputType": "text",
      "regEx": "^[^\\s@]+@[^\\s@]+\\.[^\\s@]+$",
      "storeKey": "user_email",
      "onValid": "next_node"
    },
    "logic_gate": {
      "type": "logic",
      "condition": "user_email",
      "map": {
        "admin@test.com": "admin_dash",
        "default": "user_dash"
      }
    }
  }
}

Property Glossary

regEx Standard JS regex. Only applied to inputType: "text".

storeKey The variable name used in the Lead Profile.

condition The variable name the Logic Gate looks for in the Profile.
